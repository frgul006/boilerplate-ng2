Okay... what just happened?

There is a property for flex items (direct descendants of elements with display: flex, remember?) called flex-grow.

flex-grow

"This defines the ability for a flex item to grow if necessary."

...

"If all items have flex-grow set to 1, the remaining space in the container will be distributed equally to all children.
If one of the children a value of 2, the remaining space would take up twice as much space as the others (or it will try to, at least)."

- Chris Coyier

The default value of flex-grow is 0 - which is equivalent of saying: "Don't take up more space than you need to."

Remember the structure?

-html
--body (display: flex)
---header
---main
---footer

So, the problem was that header, main and footer were all flex-grow: 0. Nothing was taking up that extra space!

By setting flex-grow: 1 on main, we got this:

-html
--body (display: flex)
---header
---main (takes up all extra space)
---footer
