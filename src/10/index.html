The main stuff

The Holy Grail Layout is a page with a header, footer, and three columns.

  • The center column (main content) should appear first in the HTML source.
  • All columns should be the same height, regardless of which column is actually the tallest.
  • They should have a fluid center with fixed-width sidebars.

Let's make it happen.

main {
flex-grow: 1;
display: flex;
}

article {
flex-grow: 1; }

nav, aside {
flex-grow: 0;
flex-basis: 25rem;
flex-shrink: 0;
}

nav {
order: -1;
}

What's happening here?

We are starting with telling the main element that aside from being a flex item (which is implied by the flex-grow property) - we want it to be a flex container.

Next we are telling the article element (a flex item of main) to take up all the white-space available

After that we are telling the nav and aside elements that they should be fixed. Never shrink, never grow. Always be at the specified flex-basis (or width).

Flex-basis???

At this point, you are probably wondering what the hell flex-basis is - and what's wrong with just using width?

The short answer is: Nothing! The default value for flex-basis is auto - which is like saying "Just look at my width and figure it out"

So what's the difference?

flex-basis is basically "width before remaining white-space is distributed between flex items". This means flex-basis is more of a guideline than an actual rule... (Get the reference?)

"This defines the default size of an element before the remaining space is distributed. It can be a length (e.g. 20%, 5rem, etc.) or a keyword."

...

"If set to 0, the extra space around content isn't factored in. If set to auto, the extra space is distributed based on its flex-grow value."

- Chris Coyier

Get on with it!
